# Submission Copy

## Name

Tarry

## One-Liner

Tarry is a robot-powered physical context layer for startup teams.

## Short Description

Tarry turns office conversations, whiteboards, and room activity into searchable team memory. Using a Reachy Mini as an embodied office companion, Tarry sees people, reacts in the room, extracts decisions and context, and writes the important moments into GBrain for later retrieval.

## Problem

Startup teams generate some of their highest-value context in the physical office: debates, whiteboard sketches, customer-call prep, pricing decisions, disagreements, and small moments of insight. Digital systems capture Slack, docs, calendar, GitHub, and CRM, but they miss what happens in the room.

## Solution

Tarry adds a physical input layer to team memory. It uses vision, audio, and robot embodiment to observe the room, make attention visible, structure context into cards, and store that context in GBrain so the team can search it later.

## Demo Promise

In the demo, Tarry watches two teammates discuss a startup decision, detects people in the room, reacts with robot movement, captures context cards, writes them into GBrain, and later retrieves the physical-room context for meeting prep.

## Why Now

AI memory is getting powerful, but most context pipelines are still digital-only. Multimodal models, local robot interfaces, and memory systems like GBrain make it possible to connect the physical office to searchable team intelligence.

## Technical Highlights

- Reachy Mini as an embodied office companion.
- Live camera path with person/face detection.
- Dashboard showing robot POV, context cards, transcript, memory writes, and retrieval.
- GBrain-backed physical-room memory source.
- ZeroEntropy semantic embeddings for retrieval.
- Replay-safe demo path when hardware or networking flakes.

## Closing Line

Every team has a digital brain. Tarry gives the office a memory.
